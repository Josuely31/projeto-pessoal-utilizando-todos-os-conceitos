Pilares de POO (Programação Orientada a Objetos)

Abstração

Abstrair é eliminar detalhes desnecessários para que possamos apenas focar. nos aspectos que são necessários para o contexto ou sistema que estamos desenvolvendo.

Por exemplo, digamos que você seja estudante e também funcionário. Para o sistema escolar eles só se preocupam com você como aluno, não se preocupam com sua propriedade como funcionário, então o sistema escolar apenas abstrai as propriedades nas quais está interessado de você tais como: nome, nota e qualificações Mas o seu sistema de trabalho está interessado nas suas propriedades como funcionário, que podem ser: número_dofuncionário, cargo e salário.

Encapsulamento

Ele se concentra em ocultar detalhes que não são relevantes para o exterior É o agrupamento de dados (propriedades) e os métodos que atuam nessas propriedades portanto, o acesso a esses dados é restrito fora do pacote (classe). Na POO, isso significa que um objeto armazena seu estado de forma privada, e apenas os métodos do objeto têm acesso para alterá-lo.

Por exemplo, se um usuário de motocicleta deseja dar partida na motocicleta; então existe uma interface para fazer isso; e o usuário não tem acesso direto para ativá-lo da própria fiação interna (sabendo que isso pode dar errado), Poderíamos até concluir que o usuário da motocicleta Você não se importa como o sistema de ignição funciona, Você só está interessado em usá-lo para começar a se mover. Assim, os detalhes da operação permanecem ocultos ao usuário, mas em vez disso você receberá uma interface fácil de usar (se necessário).

Herança

>É a forma como uma classe (filho) herda seus métodos e atributos de outra classe (pai). A herança permite a reutilização de código de uma classe em outra e é ideal para implementá-lo quando os objetos têm semelhanças entre si.

Por exemplo, um livro tem título e autor, e por outro lado uma história em quadrinhos também tem título e autor, Em OOP poderíamos usar herança para fazer o Comic herdar do Book e evitar a repetição de código desnecessário.

Polimorfismo

Refere-se à capacidade de realizar a mesma ação de maneiras diferentes.

Por exemplo, se tivermos uma classe A com método M e uma classe B que herda de A, a classe B você pode usar o mesmo método M da classe A, mas com uma realização (um pouco) diferente.

Polimorfismo é uma palavra grega que significa "com muitas formas"

Terminología POO

Objetos

Eles são a peça central do OOP. Objetos são compostos de dados (propriedades) e métodos (funções) que operam nesses dados.

Propiedade

É um dado que representa uma característica de um objeto. Uma propriedade é uma variável. Um objeto pode ter muitas propriedades.

Método

É uma função que nosso objeto pode executar. Um objeto pode ter muitos métodos.

Clase

Eles funcionam como modelos e são usados ​​para instanciar (criar) objetos. Uma classe encapsula (contém) todas as propriedades e métodos que os objetos armazenarão posteriormente. instanciado. Uma classe representa um tipo de objeto; exemplo: Livro, Carro, Cachorro.

Constructor

É um método que é chamado no momento da criação de instâncias (objetos). Construtores são úteis para (desculpe a redundância) "construir" ou inicializar propriedades dos objetos.

Getter

Um getter é um método que permite acessar uma propriedade privada de um objeto em vez de usar a propriedade diretamente.

Setter

Um setter é um método que permite acessar e modificar uma propriedade privada de um objeto em vez de modificar a propriedade diretamente.
